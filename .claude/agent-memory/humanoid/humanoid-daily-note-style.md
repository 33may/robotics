Humanoid daily notes should combine detailed continuity with concise structure. Start by naming the concrete artifact or outcome created that day, especially project documents, Linear tasks, command files, or workflow changes. Then explain why it matters in the user's casual first-person working style.

**Why:** The user liked the detailed continuity of the first daily draft but wanted the clarity of the second draft. They want dailies to be useful later, not just a vague summary.

**How to apply:** For humanoid dailies, use normal Markdown sections and tables. Prefer this shape when enough happened: `## <day theme>`, artifact/reference links near the top, `## Files changed today`, planning/task tables, decisions/gotchas, and carry-forward tasks. Use Obsidian wikilinks for vault files and absolute paths for files outside the vault. Keep prose clear and not overly official; preserve context and decisions so future sessions can resume quickly. Link this with [[humanoid-linear-task-flow]] and [[humanoid-folder-organization]].
