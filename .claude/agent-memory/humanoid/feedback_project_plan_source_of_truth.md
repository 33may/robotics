The humanoid **project plan doc** (`humanoid/docs/Initial Humanoid Project Plan.md`)
is the strategic source of truth; **Linear is only the working/execution layer**. When
planning a new block or direction, reconcile against the plan doc first — don't plan off
the board alone.

**Why:** 2026-07-01, planning block 2, the Linear board still showed Week 1–2 onboarding
tasks and had drifted from the plan's Week 3–4 semantic-perception milestone. Anton:
*"check the project plan, linear is just working plan."* The plan named semantic 3D
perception as the #1 direction and the demo critical path; the board was lagging it.

**How to apply:** In `/manager` or any planning, read the plan doc first, then align
Linear to it. Treat board drift as expected — the plan wins on strategy; the board is
just where committed, clarified tasks land. Related: [[humanoid-summer-2026-plan]],
[[humanoid-linear-task-flow]].
