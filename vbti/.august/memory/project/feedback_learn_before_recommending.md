---
name: Learn before recommending
description: Don't make strong recommendations or reframe project priorities without deeply understanding current state first
type: feedback
---

Don't jump to opinionated recommendations (like "you need RL") without first understanding the full project state and team structure.

**Why:** User has collaborators handling different domains (e.g., colleague doing RL). Making uninformed suggestions wastes time and feels presumptuous. The 90% baseline being "cherry-picked" and RL being someone else's job were both facts I should have discovered before speaking.

**How to apply:** When starting a session or encountering new context — read experiment notes, session logs, and ask clarifying questions before proposing strategy. Present what you learned, not what you think should happen.
