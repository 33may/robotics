On 2026-05-26, the project plan was rewritten in `vbti/oficial/docs/portfolio/Project Plan 26-05 update.md` to reflect the current project state.

The new framing presents the graduation project as a practical robot-learning workflow that combines real-world demonstrations, simulation infrastructure, modern VLA models, and real-robot evaluation. It no longer frames simulation as the single solution; simulation is one part of a broader workflow alongside data-efficient models, structured data pipelines, and evaluation.

Key accepted direction:
- main research question: how a practical robot-learning workflow can combine real-world demonstrations, simulation infrastructure, and modern VLA models to achieve reliable and data-efficient physical robot manipulation;
- core phases: simulation infrastructure proof of concept → real robot setup and baseline discovery → VLA training pipeline and reliable baseline → validation/data efficiency/research extensions;
- validation framing: the project sits between research and development, so the main stakeholder value is not only a working prototype but also validated knowledge, reasoning, technical decisions, and stakeholder alignment;
- current evidence path: real-world success rate, data-efficiency experiments, training-duration comparisons, qualitative videos, failure analysis, stakeholder feedback, and simulation-pipeline evidence for future sim-real/RL-generated data.

**Why:** This is the current planning baseline for graduation report/table-meeting discussions and supersedes older simulation-first project-plan language.

**How to apply:** When updating report material, use `Project Plan 26-05 update.md` as the source of truth for project-plan wording. Keep simulation important but position it as one component in the larger robot-learning research workflow. Link with [[project-state-2026-05-26]].
