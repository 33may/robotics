## Problems

So the problems are:

1. The model is not generalized, but overfilled on actions, so we need to understand how much of the demonstration could actually produce the same results,
2. Since the model only seen the fixed cup position, it is conditioning the grabbing the duck action on the cup position, which is completely irrelevant for the given task.

So lets thing what we could do, the easiest thing to consider for working with the first problem, it to just sub-sample the new smaller versions of the datasets from the 03-merged dataset. Train the model on each of them and check in which time the model could provide the good results. In order to make the good evaluation we need to define the good evaluation protocol in real world and also create the evaluation scenario in simulation(for that we need to actually check is better model in sim eval actually corresponds to better model in real, since we are aiming to optimize real world performance).

The second thing that is way more crutial is to understand what we could do to fix the problem with the cup. Now the model is really good if the cup remains at the same position as it was in the dataset, however if we move the cup to other position, the grabbing duck performance decrease significantly, and of course the model can't put the duck and reach the cup, since it has never seen other cup positions, so it moves to the middle, actually it might be partially solved if we decide to predict the next delta action instead of joint positions, but I could imagine that  the model will become way less stable.

![[Pasted image 20260403140342.png]]

**Idea 1 (left side):** The red blobs on the camera images are masks applied directly to the raw images before the frozen encoder. Random masking on pixel space.

**Idea 2 (center-right):** The red x's in the blue embedding boxes are dropout applied to the vision embeddings after the frozen encoder, before the action head.

So now the question is which regularization is better for your case, given that cup variation in the dataset is already planned.

My take on the comparison:

**Idea 1 (image masking):** Risky with frozen encoder as we discussed — SigLIP wasn't trained on masked images. But it has one advantage: it forces spatial robustness. If the cup region is randomly occluded, the encoder literally can't extract cup features from those patches.

**Idea 2 (embedding dropout):** Clean, standard, no OOD risk for the encoder. But as I said — cup info is distributed across many dimensions, so random dropout might not reliably break the correlation.

Actually — now that you're saying you'll have cup variation in the dataset anyway, the framing changes. You're not asking "which one fixes cup conditioning" — you're asking "which regularization makes the model learn better from varied data." In that case I'd lean toward idea 2. It's the safer bet, it works with the encoder as designed, and combined with actual cup variation in data, the model should learn to ignore cup position naturally.

Is that how you're thinking about it — regularization on top of better data, not instead of it?
